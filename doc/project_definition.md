# 프로젝트 정의서

## 1. 프로젝트 명칭
- SilentTalk(가칭)

 
## 2. 개발 시스템 명칭
- AI 기반 상황 인식 무음 커뮤니케이션 지원 시스템


## 3. 개발 시스템 설명
- 본 시스템은 청각장애인 및 난청 사용자를 대상으로, 카메라 기반 상황 인식과AI를 활용하여 의사소통을 지원하는 서비스이다. 사용자의 현재 위치 및 환경을 분석하여 상황에 적합한 문장을 자동으로 추천하고, 버튼 기반 인터페이스를 통해 빠르게 의사표현을 할 수 있도록 한다.
이를 통해 타이핑 중심의 느린 의사소통 문제를 해결하고, 자연스러운 대화 흐름을 유지하는 것을 목표로 한다.


## 4. 배경 및 필요성
### 청각장애 인구 규모
- 세계보건기구에 따르면 전 세계 약 15억 명이 청력 손실을 경험하고, 이 중 4억 3천만 명은 재활이 필요한 수준의 난청을 가진다고 한다. 또한 2050년까지 약 25억 명이 청력 문제를 겪을 것으로 예상한다.

### 국내 청각장애 현황
- 보건복지부 및 한국장애인고용공단 자료 기준, 한국 청각장애인은 약 40만 명 이상이며 난청 포함 시 훨씬 더 많은 인구가 존재할 것으로 얘상된다. 고령화로 인해 지속적으로 증가 중이다.

### 본 프로젝트는 전 세계 15억 명 이상의 청력 손실 인구와 국내 약 40만 명 이상의 청각장애인이 겪는 의사소통 문제를 해결하기 위해 제안되었다. 기존의 타이핑 기반 의사소통 방식은 일반 대화 대비 속도가 느리며, 상황을 고려하지 못하는 한계를 가진다. 이에 따라 본 프로젝트는 AI 기반 상황 인식을 통해 즉각적인 의사 표현을 지원하는 시스템의 필요성에서 출발하였다.


## 5. 개발 시스템 핵심 기능
(1) 상황/장소 인식 (Computer Vision)
- 카메라 입력을 통해 카페, 병원, 학교 등 환경 자동 인식
- 상황에 맞는 문장 추천
(2) 버튼 기반 초고속 의사 표현
- 타이핑 없이 버튼 선택으로 즉시 의사 전달
(3) AI 기반 대화 흐름 추천
- 대화 맥락을 분석하여 다음 문장 자동 추천
(4) 상황 맞춤 문장 자동 생성 (NLP)
- 장소 + 상황에 맞는 자연스러운 문장 생성
(5) 제스처 기반 입력 기능 (Computer Vision)
- 손동작으로 문장 선택 및 UI 조작
(6) 사용자 맞춤 학습 기능
- 자주 사용하는 문장을 학습하여 우선 추천


## 6. 기존 유사 소프트웨어 조사 및 분석
(1) Google Live Transcribe
- 기능
  * 음성을 텍스트로 변환하여 제공
- 장점
  * 높은 정확도
  * 실시간 처리 가능
- 한계
  * 상황 인식 기능 없음
  * 사용자 의사 표현 지원 부족
(2) Ava
- 기능
  * 그룹 대화 자막 제공
- 장점
  * 화자 구분 가능
  * 다수 사용자 지원
- 한계
  * AI 기반 상황 추천 기능 없음
  * 버튼 기반 빠른 의사 표현 미지원
(3) Speech Assistant AAC
- 기능
  * 문장 선택 기반 의사소통 지원
- 장점
  * 버튼 기반 인터페이스 제공
- 한계
  * 상황 자동 인식 없음
  * 사용자 직접 선택 필요

#### 기존 서비스는 텍스트 전달 중심, 본 프로젝트는 상황 이해 + 즉각 반응 중심
